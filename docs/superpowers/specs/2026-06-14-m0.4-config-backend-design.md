# P2-M0.4 — Config backend: operator.yaml read/update API + live hot reload

## Context

M0.2 (privacy posture) and M0.3 (off-origin widget) both deferred runtime
tunability to "the config backend." M0.3 says verbatim: *"Admin-tunable later
(M0.4 config backend + M0.6 modal); M0.3 = operator.yaml + defaults."* Today the
only ways to change a non-secret setting are hand-editing `operator.yaml` and
restarting, or setting a `NOVA_*` env override — neither is reachable from the
admin SPA, and there is no programmatic read of the *effective* config.

M0.4 delivers the **backend** the future setup wizard (M0.5) and admin settings
modal (M0.6) call: an operator-only API (and `novactl` parity) to **read** the
effective config and **update** it, persisting to the canonical `operator.yaml`
and **hot-reloading** in process the fields that can be applied live. No UI ships
in M0.4 — it is API + CLI + persistence + hot-reload only.

The load-bearing constraint: runtime config is currently **frozen into closures
and channel semaphores at boot** (`newAdmission` sizes a buffered channel once;
`storage` sizes its `assembly` channel once via `WithWriteLimits`;
`middleware.CORS` captures a frozen origin set; upload caps are one-shot). A
buffered channel's capacity cannot be resized, so the two channel semaphores must
become **atomic counters** to be live. Everything else either reads a snapshot
per-use or is honestly classified `restart`.

## Decisions (ratified with Bug, 2026-06-14)

- **operator.yaml stays the single canonical non-secret store.** Updates atomically
  rewrite it and re-run the existing validation path. **No DB settings table** for
  config values — one source of truth, operator can still hand-edit, and it matches
  the M13 ROADMAP ("operator.yaml is the canonical non-secret config source, with
  `NOVA_*` env reads preserved as overrides").
- **Live in-process hot reload** for fields that can be hot-applied; no coordinator
  restart for those.
- **Full operator.yaml surface is editable + validated + persisted**, but
  **editable ≠ hot-reloadable**. Every field carries an **effect classification**
  — `live` | `restart` | `env-only-inert` — surfaced in the read API, so "full
  surface" never misleads an operator about what actually took effect.
- **Env overrides still win at runtime.** A PATCH persists the yaml value (records
  operator intent even when shadowed), but the value applied to the live snapshot
  is `yaml ⊕ env`. The read API marks env-shadowed fields `source:"env"`.
- **No new secrets in scope.** `operator.yaml` holds only non-secrets (paths, not
  inline secrets); secrets continue to resolve via `config.ResolveSecret` from
  `/run/secrets`/env and are never read, written, or returned by this API.
- **Single validation code path.** Updates validate through the same
  `config.LoadFromBytes` (→ `validate` + `ApplyPrivacyPreset` + `applyUploadDefaults`)
  the loader and setup renderer already use. No forked validator.

## Components

### 1. Atomic config snapshot — `internal/config/reload` (new package)
A `Store` is the live source of effective config. New package (not `internal/config`)
because `internal/api/middleware`, `internal/api/handlers`, and
`pkg/coordinator/storage` all import `internal/config`; the holder must be importable
by all of them without an import cycle.

```go
type Store struct {
    ptr      atomic.Pointer[config.Config] // effective cfg (post validate/preset/defaults ⊕ env)
    mu       sync.Mutex                     // serializes writers (LWW / If-Match guard)
    version  atomic.Uint64                  // bumped on each successful swap; the etag source
    envPins  map[string]struct{}            // keys pinned by NOVA_* env at boot (re-overlaid on Swap)
    subs     []func(old, new *config.Config)
}

func New(initial *config.Config, envPins map[string]struct{}) *Store
func (s *Store) Load() *config.Config                       // lock-free per-use read
func (s *Store) Version() uint64
func (s *Store) Subscribe(fn func(old, new *config.Config)) // wiring-time only
func (s *Store) Swap(next *config.Config) uint64            // store + bump + fire subs under mu
```

Constructed in `cmd/coordinator` from the effective config; threaded into
`coordinator.Config` as `ConfigStore *reload.Store`. **A nil store ⇒ today's static
behavior** at every consumer, so all existing call sites and tests are unchanged.

### 2. Refactors that make `live` fields hot-swappable
- **CORS** → `middleware.CORSReloadable(store)`: per request reads `store.Version()`;
  rebuilds a cached `*compiledCORS` (allowed-origin set + pre-joined method/header
  strings) only when the version changed — so the map/joins rebuild once per *config
  change*, never per request. `middleware.CORS(config.CORS)` stays for the static path.
- **admission global + per-session** → channel→counter rewrite. Replace
  `global chan struct{}` with `globalInUse int` guarded by the existing `mu`;
  `TryAcquire` reads the current `MaxConcurrentGlobal` / `MaxConcurrentPerSession`
  from `store.Load()` at call time. Preserves the non-blocking try, the
  global-rollback-on-per-credential-failure, and `<=0 ⇒ unbounded`. Lowering a limit
  below current in-flight is safe (new acquires refused until it drains).
- **assembly concurrency** → `assembly chan` → `assemblyInUse atomic.Int64` +
  `assemblyLimit atomic.Int64`; acquire becomes `Add(1)` / compare / `Add(-1)`,
  preserving the exact non-blocking `ErrServerBusy`. `storage.SetWriteLimits` updates
  the limit live; subscribed to the store.
- **max-upload-size** → `storage.maxUploadSize` becomes `atomic.Int64`; `UploadHandler`
  reads it live from the store.
- **max-files-per-session** → `UploadHandler` reads `store.Load()…MaxFilesPerSession`
  per `CreateTus`.

### 3. Update flow (one validation path; nothing persists on failure)
PATCH (partial merge) is primary; PUT (full replace) is the power path.
1. **Re-read on-disk** config via `LoadFromFile` so an out-of-band hand-edit is the
   merge base.
2. **Merge** the request — map-based deep merge (marshal base to a map, deep-merge the
   request, unmarshal back) to preserve tri-state `*bool` and `omitempty` semantics
   (notably `coordinator.record_source_ip`).
3. **Validate + normalize** via `config.LoadFromBytes(merged)` → `validate` +
   `ApplyPrivacyPreset` (returns warnings) + `applyUploadDefaults`. Failure → **422**,
   persists nothing.
4. **Atomic write** — new `config.WriteAtomic(path, *Config)`: `yaml.Marshal` →
   round-trip self-check (`LoadFromBytes` must re-accept it, mirroring `render.go`) →
   write `path+".tmp-<rand>"` (0644) → `fsync` → `os.Rename` → `fsync` parent dir.
   **This temp+rename discipline does not exist today** (`setup/commit.go` uses a
   plain `os.WriteFile` with only sentinel-last ordering); M0.4 introduces it.
5. **Swap** — `store.Swap(validatedYaml)`: `Swap` itself re-overlays the boot-time
   `envPins` (env still wins live) before storing, bumps version, and fires
   subscribers (the storage `SetWriteLimits` updater). Admission reads its limits
   live at `TryAcquire`, and CORS / the upload handler read lazily on the next
   request — so those need no subscriber.

### 4. API surface (operator-only, admin vhost, under `/api/v1/admin`)
Wired exactly like `upload_tokens_admin` — `ConfigAdmin *handlers.ConfigAdminHandler`
on `api.ServerConfig` (nil ⇒ 404), guarded by `bearer.RequireRole("operator")`.

- `GET /api/v1/admin/config` → `{config, version, privacy_warnings, fields}` where
  `fields[<dotted.path>] = {effect, source, shadowed_by_env}`; `Cache-Control: no-store`.
- `PATCH /api/v1/admin/config` → partial merge; **200** returns the new effective
  state + `restart_required: [...]` (changed fields whose effect is `restart` /
  `env-only-inert`); **422** `config_invalid` on validation failure; optional
  `If-Match: <version>` → **409** `config_conflict` when stale (optimistic
  concurrency; absent ⇒ last-writer-wins over a disk-re-read base).
- `PUT /api/v1/admin/config` → full replace (same response contract).

**Audit:** inject `*auditlog.Writer` via `SetAuditWriter` (as `signing_admin.go` does);
on each successful write, best-effort `Write` an `Action:"config.updated"` entry with
an old→new field diff + `version` + `restart_required` (post-persist; never fails the
request). The effect/source map is a package-level static classification table keyed
by dotted path, plus the boot-time env-pin set.

### 5. novactl surface — `cmd/novactl/config.go`
Mirrors `upload_token.go` (stdlib flag subcommands, HTTP-only, creds from
`~/.config/nova/credentials.json`); dispatched via `case "config"` in `main.go`.
- `novactl config get [--effects]` — GET; pretty-print; `--effects` renders the
  per-field effect/source table.
- `novactl config set <dotted.path> <value> [--json]` — builds a one-field PATCH;
  on 200 prints and, if non-empty, **warns** the `restart_required` list.
- `novactl config apply --config-file <p>` — read a YAML/JSON doc → PUT full replace.
Adds a small `patchJSON` helper beside `postJSON`; operator-only error messages
identical to the upload-token commands.

### 6. Effect classification (the full table; surfaced in GET `fields`)
| Section / field | Effect | Reason (verified consumption site) |
|---|---|---|
| `uploads.cors.*` | **live** | only consumer is `middleware.CORS`; compiled-snapshot cache |
| `uploads.limits.max_concurrent_global` | **live** | `admission.global` channel → counter |
| `uploads.limits.max_concurrent_per_session` | **live** | `admission.max`, read at `TryAcquire` |
| `uploads.limits.max_files_per_session` | **live** | read at `CreateTus` |
| `uploads.max_upload_size_bytes` | **live** | `UploadHandler` + `storage` size checks (atomic) |
| `uploads.max_concurrent_assembly` | **live** | `storage.assembly` channel → counter |
| `uploads.session_ttl_seconds`, `uploads.tmp_dir` | **restart** | bound into `upload.NewStore` at boot |
| `uploads.public_uploads` | **restart** | gates route mounting in `server.go` |
| `coordinator.record_source_ip` | **restart** | captured in `UploadHandler.recordIP`; tangled with preset resolution |
| `coordinator.public_ipfs_dht` | **restart** | consumed at Kubo backend boot |
| `auth.*` (issuer_url, client_id, paranoid, anonymous, role_mapping…) | **restart** | verifiers/issuer + paranoid preset built once at boot |
| `tls.*` | **restart** | TLS is nginx-terminated (two-vhost); coordinator never reads it live |
| `operator.*`, `tos_url`, `moderation.*`, `source_ip_retention_days`, `webhooks` | **restart** | validated + persisted now; consumed at boot / by subsystems built in `coordinator.New` |
| `orchestrator.*` | **env-only-inert** | per M13 deferral; no runtime consumer reads the yaml section today |
| `federation.*` | **env-only-inert** | no consumption site (Phase-1 single-node); forward-compat persist only |
| `integrity_audit.*` | **env-only-inert** | runtime uses `integrity.DefaultCadences()` constants + `NOVA_INTEGRITY_AUDIT_ENABLED` |
| `signed_urls.*` | **env-only-inert** | read from `NOVA_SIGNED_URL_*` env, never the yaml section |
| `master_key_rotation.*` | **env-only-inert** | read from `NOVA_MASTER_KEY_REWRAP_*` env, never the yaml section |

`env-only-inert` = part of the editable/validated/persisted surface (yaml stays
canonical so a future milestone can wire it), but **no runtime consumer reads it
today** — a PATCH changes the file and validates but has no live or restart effect
until that wiring lands. GET marks these explicitly so an operator is never misled.

## Security notes
The config surface holds **no secrets** — the struct exposes only `*_file`/`*_path`
fields (paths). A `write_test.go` assertion + a handler invariant comment forbid any
inline-secret field from entering yaml or the API response. The endpoints are
**operator-only** (`RequireRole("operator")`), on the admin vhost. Every write is
audit-logged with an old→new diff for after-the-fact review. Validation runs *before*
the atomic write, and temp+rename means readers never observe a partial file, so a
rejected or crashed write leaves the prior config intact. Env-shadowed fields are
flagged so an operator cannot be fooled into thinking a persisted-but-shadowed change
took effect. Hot-reload swaps are atomic (single `atomic.Pointer.Store`); a
subscriber panic is recovered and logged with the file already persisted and the
pointer already swapped, so the system stays consistent (worst case a limit lags one
change).

## Verification
- **Go:** `WriteAtomic` round-trip + tri-state preservation + no-temp-leftover +
  secret-free; `reload.Store` concurrent Load/Swap + monotonic version + subscriber
  fan-out; storage/admission counters preserve `ErrServerBusy`/429 and honor a limit
  changed at runtime; CORS reloadable flips allowed-origins on version change without
  rebuilding the router; `ConfigAdminHandler` GET (effect/source incl. env-shadow) +
  PATCH/PUT (merge→validate→persist→swap; 422 persists nothing; audit diff;
  `restart_required`; `If-Match` 409). Integration tests use the `dbtest.New`
  testcontainers Postgres + `httptest` pattern. `gofmt`; `go build ./...`.
- **Targeted integration:** a `uploads.cors` PATCH changes allowed-origin behavior on
  the **next in-process request** (live); an `auth.issuer_url` PATCH persists to yaml +
  reports `restart_required` and does **not** change live auth.
- **Optional manual smoke:** `novactl config get --effects` → `config set
  uploads.limits.max_concurrent_global 8` → re-`get` shows the version bump + live
  effect; `config set auth.issuer_url …` warns restart-required.
