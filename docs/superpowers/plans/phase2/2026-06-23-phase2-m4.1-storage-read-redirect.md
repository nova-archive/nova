## Goal

Turn the coordinator's local Kubo from *canonical origin* into a *bounded,
prunable cache*; make the **verified donor replica set the durable substrate**,
without changing Nova's donor-blind, coordinator-served trust model. Deliverables:
donor-backed reads on cache miss (select sourceable holder → fetch ciphertext →
verify → decrypt → serve → re-cache), `require_replication_quorum_before_commit`
(async staging), origin/staging pruning with `prune_safety_floor`,
`coordinator_storage_mode` (`origin_copy`/`bounded_cache`/`transient`) + bounded
hot cache, transform re-fetch.

## Architecture / tech stack

Go (go.mod 1.26.2). Coordinator: chi, pgx/sqlc, embedded Kubo (`internal/ipfs`),
`pkg/coordinator/storage` read/write core. Donor (`cmd/node`): dependency-light
agent + new inbound mTLS read-source server, Kubo **sidecar** over loopback
(`internal/node/ipfsclient`). Federation: mTLS-over-Nebula, Ed25519 grants
(`internal/federation/{wire,tokens}`), donor-safe `wire`. Durable state: Postgres
goose migration `0013`. Config: static `internal/config` + hot-reload
`reload.Store` + admin `/settings`.

## Task 1 — Envelope-size propagation (D-M4.1-16)

Prerequisite correctness fix; also patches a latent M4 defect (preflight/grants
reason about plaintext size while serving the ciphertext envelope). Make
`envelope_size` flow through **every** grant/expectation path.

**SQL (`internal/db/queries/federation.sql`)** — switch the two plaintext-sized
selects to envelope size:

**Preserve each query's existing state filter — do not silently broaden
eligibility.** `GetBlobSize` (AssignPin) has **no** state filter today, so keep it
unfiltered; `GetBlobByteSize` (source preflight) filters `state='active'`, so its
envelope-size replacement must keep that filter.

```sql
-- name: GetBlobSize :one   (feeds AssignPin → pin_changes.byte_size; no state filter, as today)
SELECT m.envelope_size FROM blob_manifests m WHERE m.cid = $1;

-- name: GetEnvelopeSize :one   (source preflight + read tier; preserves the active filter)
SELECT m.envelope_size
FROM blob_manifests m JOIN blobs b ON b.cid = m.cid
WHERE m.cid = $1 AND b.state = 'active';

-- name: GetPinSnapshotPage :many   (donor desired-assignment size)
SELECT pa.cid, pa.assignment_id, pa.generation, m.envelope_size AS byte_size, pa.assigned_at
FROM pin_assignments pa
JOIN blob_manifests m ON m.cid = pa.cid
WHERE pa.node_id = $1 AND pa.cid > $2
ORDER BY pa.cid
LIMIT $3;
```

`GetEnvelopeSize` replaces `GetBlobByteSize` in the source preflight. `mintSource`
already maps
`MaxBytes: ch.ByteSize`, so once `pin_changes.byte_size` is envelope-sized the
token `max_bytes` is correct with no further change. Donor `progress.ByteSize`
means envelope size from M4.1 on (documented; no migration needed — it is donor
JSON).

**Old rows.** Pre-M4.1 `pin_changes.byte_size` rows hold plaintext size; a
generation bump refreshes them, but to avoid spurious `max_bytes` refusals on
existing assignments, migration `0013` (Task 4) backfills
`UPDATE pin_changes c SET byte_size = m.envelope_size FROM blob_manifests m WHERE
m.cid = c.cid AND c.byte_size <> m.envelope_size`. (The donor transfer is also
bounded by `max_transfer_bytes`, so this is defense-in-depth.)

**Files:** `internal/db/queries/federation.sql`,
`internal/federation/coordinator/blob.go` (preflight → `GetEnvelopeSize`),
regenerate sqlc.

## Task 12 — Pruner + `prune_safety_floor` + crash recovery (D-M4.1-12)

**Config:** `PruneSafetyFloor int`; `PrunerInterval`. **`storage/prune.go`** loop:

```
candidates := ListPruneCandidates(mode-eligible classes)   // committed, local_present
for c in candidates:
  // crash-window reconcile FIRST (Bug #4): align projection with reality
  has := backend.Has(c.cid)
  if c.local_present && !has: SetLocalPresence(absent); continue           // unpin already happened
  if !c.local_present && has && c.local_role==absent: backend.Unpin OR mark cache per mode
  n := CountSourceableHolders(c.cid, stale)
  if n >= PruneSafetyFloor:
     backend.Unpin(c.cid); SetLocalPresence(local_present=false, role per mode); emit storage.prune.applied{cid,holders=n,bytes_reclaimed}
  else:
     if c.local_present: keep/re-pin (retain); emit storage.prune.skipped_floor{cid,holders=n}
     else: emit storage.prune.below_floor_alert{cid,holders=n}; stop pruning this cid/class; optional emergency re-fetch into cache (Task 7); cache NEVER counts as a replica
bounded_cache also LRU-evicts past BoundedCacheMaxBytes among holder-safe blobs.
```

Two crash windows are explicitly handled by the `backend.Has` reconcile step
before any prune decision (projection is the storage-policy authority, so it must
be re-aligned to the blockstore on each pass).
