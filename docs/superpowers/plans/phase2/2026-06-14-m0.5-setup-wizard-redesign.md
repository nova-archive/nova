# Setup-Wizard Redesign (M0.5) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the first-run `web/setup` wizard so it explains its load-bearing privacy/security choices at decision time — consequence copy, "learn-this vs abstract-away" jargon info-buttons, and a tri-state `paranoid` parent toggle over individually exposed privacy constituents — backed by a small additive `setup.Answers` expansion so the new toggles reach `operator.yaml`.

**Architecture:** Three new presentational React components (`InfoTerm` native-`<details>` disclosure, `ConsequenceNote`, tri-state `ParanoidSection`) plus a `glossary.ts` term table drive the redesign; the wizard's `FormState`/`toAnswers` gains three protective-state booleans that map to explicit `record_source_ip` / `source_ip_retention_days` / `public_ipfs_dht` values plus a derived `paranoid`. The Go side gains three additive `omitempty` `Answers` fields mapped through `RenderOperatorYAML`; omitted fields preserve today's behavior, so older callers and `novactl setup` are unaffected.

**Tech Stack:** TypeScript + React 18, Vite 8 / Vitest 4 + `@testing-library/react` (web); Go stdlib `testing` + `gopkg.in/yaml.v3` (backend). CSS Modules. No new dependency.

Spec: `docs/superpowers/specs/2026-06-14-m0.5-setup-wizard-redesign-design.md`. Branch: `p2-m0.5-wizard-redesign`. Single tag at end: `p2-m0.5-wizard-redesign`. No remote push; ff-merge + annotated tag per the milestone workflow.

**Cross-cutting conventions (apply every task):**
- Go: run `gofmt -w` on touched files before commit (only files you touched). No DB schema change in this milestone — do **not** run `make sqlc-generate`.
- Web: tests are `*.test.tsx` co-located in `web/setup/src/wizard/`; run from `web/setup/` with `npm test`. Style mirrors the existing `Wizard.orientation.test.tsx` (`vitest` + `@testing-library/react`, `afterEach(cleanup)`).
- Go tests in `internal/setup` are stdlib (`package setup`, `t.Fatalf`), reuse the `validAnswers()` helper in `answers_test.go`. No testify in this package.
- Commit messages: conventional + `(P2-M0.5)`, trailing `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Backward-compat invariant:** every new `Answers` field is `omitempty`; a payload that omits all three must render byte-identical config to today (verified in Task 1).

`web/setup/src/wizard/InfoTerm.tsx`:

```tsx
import type { ReactNode } from 'react'
import s from './wizard.module.css'
import { GLOSSARY } from './glossary'

// InfoTerm renders a "learn-this" jargon term: the term text (dotted underline)
// followed by a native <details> ⓘ disclosure that expands a plain-English
// explanation. Native <details> is keyboard- and screen-reader-accessible by
// default — the <summary> carries the accessible name and the glyph is
// decorative (aria-hidden). Unknown ids render as plain text (no button).
export function InfoTerm({ id, children }: { id: string; children: ReactNode }) {
  const entry = GLOSSARY[id]
  if (!entry) return <>{children}</>
  return (
    <span className={s.infoTerm}>
      <span className={s.infoLabel}>{children}</span>
      <details className={s.infoDetails}>
        <summary className={s.infoSummary} aria-label={entry.label}>
          <span aria-hidden="true">ⓘ</span>
        </summary>
        <span className={s.infoBody}>{entry.body}</span>
      </details>
    </span>
  )
}
```

`web/setup/src/wizard/ParanoidSection.tsx`:

```tsx
import { useEffect, useRef } from 'react'
import s from './wizard.module.css'
import { InfoTerm } from './InfoTerm'
import { ConsequenceNote } from './ConsequenceNote'

// Protective state: each flag true = hardened. Mirrors operator intent; the
// wizard maps these to explicit operator.yaml values in toAnswers().
export interface ParanoidState {
  hardenNoIPRecording: boolean
  hardenShortRetention: boolean
  hardenPrivateDHT: boolean
}

const CHILD_KEYS = ['hardenNoIPRecording', 'hardenShortRetention', 'hardenPrivateDHT'] as const

export function ParanoidSection({
  value,
  onChange,
}: {
  value: ParanoidState
  onChange: (next: ParanoidState) => void
}) {
  const allOn = CHILD_KEYS.every((k) => value[k])
  const noneOn = CHILD_KEYS.every((k) => !value[k])
  const indeterminate = !allOn && !noneOn

  // `indeterminate` is a DOM property, not an attribute — set it imperatively.
  const parentRef = useRef<HTMLInputElement>(null)
  useEffect(() => {
    if (parentRef.current) parentRef.current.indeterminate = indeterminate
  }, [indeterminate])

  const toggleParent = () => {
    const next = !allOn // fully on → clear all; otherwise → set all
    onChange({ hardenNoIPRecording: next, hardenShortRetention: next, hardenPrivateDHT: next })
  }
  const setChild = (k: (typeof CHILD_KEYS)[number], v: boolean) => onChange({ ...value, [k]: v })

  return (
    <div className={s.paranoid}>
      <label className={s.toggle}>
        <input
          ref={parentRef}
          type="checkbox"
          checked={allOn}
          onChange={toggleParent}
          aria-label="Harden privacy (paranoid)"
        />
        <span className={s.lbl}>
          Harden privacy (<InfoTerm id="paranoid">paranoid</InfoTerm>)
          {indeterminate && <span className={s.partial}> — partial</span>}
        </span>
      </label>
      <span className={s.hint}>Sets every protection below. Uncheck any one to relax it.</span>

      <div className={s.children}>
        <label className={s.childToggle}>
          <input
            type="checkbox"
            checked={value.hardenNoIPRecording}
            onChange={(e) => setChild('hardenNoIPRecording', e.target.checked)}
          />
          <span className={s.lbl}>
            Don’t record <InfoTerm id="source-ip">uploader IP addresses</InfoTerm>
          </span>
        </label>
        {!value.hardenNoIPRecording && (
          <ConsequenceNote tone="warn">Each blob will store the client’s source IP.</ConsequenceNote>
        )}

        <label className={s.childToggle}>
          <input
            type="checkbox"
            checked={value.hardenShortRetention}
            onChange={(e) => setChild('hardenShortRetention', e.target.checked)}
          />
          <span className={s.lbl}>Keep IP logs 1 day, not 30</span>
        </label>
        {value.hardenShortRetention ? (
          <ConsequenceNote>Only applies while IP recording is on.</ConsequenceNote>
        ) : (
          <ConsequenceNote tone="warn">Recorded IPs are kept 30 days.</ConsequenceNote>
        )}

        <label className={s.childToggle}>
          <input
            type="checkbox"
            checked={value.hardenPrivateDHT}
            onChange={(e) => setChild('hardenPrivateDHT', e.target.checked)}
          />
          <span className={s.lbl}>
            Keep pinned CIDs off the <InfoTerm id="ipfs-dht">public IPFS DHT</InfoTerm>
          </span>
        </label>
        {!value.hardenPrivateDHT && (
          <ConsequenceNote tone="warn">This node will advertise stored CIDs publicly.</ConsequenceNote>
        )}

        <div className={s.infoRow}>
          · No outbound webhooks <span className={s.muted}>(none configured)</span>
        </div>
        <div className={s.infoRow}>
          · Metrics stay loopback-only <span className={s.muted}>(not internet-exposed)</span>
        </div>
      </div>
    </div>
  )
}
```
